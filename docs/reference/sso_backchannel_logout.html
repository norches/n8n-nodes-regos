backchannel_logout

[POST] /oauth/backchannel_logout

Backchannel Logout используется для серверного уведомления о выходе пользователя. Обычно этот метод не вызывается вручную пользователем и не нужен для простой кнопки входа. Он нужен интеграциям, которые должны завершить серверную сессию, когда REGOS сообщает о выходе.

Запрос отправляется как application/x-www-form-urlencoded и содержит logout_token. 

curl -X POST https://auth.regos.uz/oauth/backchannel_logout \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "logout_token=eyJ...logout_token"

logout_token должен быть JWT, подписанным доверенным ключом и содержащим данные сессии. После успешной обработки сервер возвращает 200 OK. Если токен отсутствует, истек или не прошел проверку, будет возвращена OAuth-ошибка.

Если вы подключаете обычный вход пользователя через REGOS, начните с /oauth/logout; backchannel-сценарий нужен только для согласованных серверных интеграций.
