События

Событие (event) - единый объект уведомления REGOS. У события есть уникальный event_id, тип (action), время возникновения (occurred_at) и полезная нагрузка (data).

Persistent-события сохраняются в истории 7 дней с момента возникновения и могут быть восстановлены через Event/Get . Transient-события актуальны только в момент доставки, не сохраняются в истории и не возвращаются через Event/Get.

Event

Название Тип данных Обязательность Описание
event_id String Обязательный UUID события. Используйте для идемпотентности и как last_event_id в Event/Get ; не используйте для сортировки
occurred_at DateTime Обязательный Дата и время возникновения события в REGOS, UTC, ISO 8601
action String Обязательный Тип события. Допустимые значения перечислены в разделе Список событий
data Object Обязательный Полезная нагрузка события. Состав зависит от action

Пример event

{
  "event_id": "2f0f7b7a-7c43-4d91-917f-8f412ad0ab5e",
  "occurred_at": "2026-06-26T10:00:00Z",
  "action": "DocOrderDeliveryStatusSet",
  "data": {
    "id": 54321,
    "status": "Completed"
  }
}

Хранение и восстановление

Persistent-события сохраняются в истории 7 дней с момента возникновения и могут быть восстановлены через Event/Get в пределах этого срока.

Если история событий ещё недоступна для аккаунта, доставка событий продолжает работать без сохранения истории. Такие события нельзя восстановить через Event/Get .

Transient-события не сохраняются в истории и не возвращаются через Event/Get . К transient-событиям относятся события, актуальные только в момент доставки, например ChatWriting и ChatSuggest.
