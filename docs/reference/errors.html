HTTP коды ответов сервера

На уровне прикладной логики REGOS:API использует единый контракт ответа: HTTP 200 + ErrorResult при ошибках.
Коды 4xx/5xx могут возвращаться внешней инфраструктурой (proxy/gateway) или при недоступности сервера до уровня API.

Код Описание
200 Сервер API ответил корректно.
400 Может возвращаться внешней инфраструктурой. На уровне API ошибки валидации возвращаются в HTTP 200 через ErrorResult.
401 Может возвращаться внешней инфраструктурой. На уровне API ошибки авторизации возвращаются в HTTP 200 через ErrorResult.
403 Может возвращаться внешней инфраструктурой. На уровне API ошибки доступа возвращаются в HTTP 200 через ErrorResult (например, 9006, 1096).
404 Может возвращаться внешней инфраструктурой. На уровне API ошибки маршрута/метода возвращаются в HTTP 200 через ErrorResult.
409 Может возвращаться внешней инфраструктурой. На уровне API прикладные ошибки возвращаются в HTTP 200 через ErrorResult.
415 Может возвращаться внешней инфраструктурой при неподдерживаемом Content-Type. На уровне API прикладные ошибки возвращаются через ErrorResult.
429 Может возвращаться внешней инфраструктурой (proxy/gateway). На уровне API прикладные ошибки лимитов возвращаются в HTTP 200 через ErrorResult (например, код 8213).
410 Может возвращаться внешней инфраструктурой. На уровне API устаревание/недоступность методов отражается в ErrorResult.
500 Может возвращаться внешней инфраструктурой или сервером до прикладного слоя API.
502 Может возвращаться внешней инфраструктурой (proxy/gateway).
503 Может возвращаться внешней инфраструктурой или при недоступности сервера.
