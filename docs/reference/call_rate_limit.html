Лимиты запросов

Чтобы сервис REGOS работал быстро и без сбоев для всех пользователей, мы ввели ограничения на частоту запросов. Лимиты действуют отдельно для каждого отдельного приложения или интеграции (connected_integration_id).
Они помогают поддерживать стабильность даже при большой нагрузке, распределять ресурсы честно между всеми клиентами и делать поведение API предсказуемым — вы всегда знаете, сколько запросов может отправить ваше приложение.

Лимиты

Что ограничиваем Значение
Средний поток запросов 2 запроса в секунду
Максимальный запас до 50 запросов подряд

Как это устроено

У каждой интеграции или приложения есть свой «счётчик запросов».

  • Счётчик пополняется со скоростью 2 запроса в секунду.
  • Можно накопить запас до 50 запросов и отправить их одним большим пакетом.
  • Если запас исчерпан, новые запросы не выполняются — пока счётчик не пополнится.

Примеры:

  • Делаем 1–2 запроса в секунду — всё работает стабильно.
  • Отправили подряд 30 запросов — тоже нормально (берутся из запаса).
  • Отправили подряд 60 запросов — выполнится только 50, а для оставшихся вернётся HTTP 200 с ErrorResult (код 8213).
